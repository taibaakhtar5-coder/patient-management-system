## Severity Definitions

* **Critical**: Causes system crash, data corruption, or completely blocks testing of a core feature.
* **High**: A major feature is broken, but a workaround exists.
* **Medium**: A minor feature is broken, or a UI element is severely misaligned/malfunctioning.
* **Low**: Cosmetic issue, spelling error, or minor visual alignment problem.

## Workflow Statuses

1. **New / Open**: Bug has been logged and verified, but not yet worked on.
2. **In Progress**: Someone is actively working on a fix.
3. **Resolved**: A fix is committed to the feature branch and awaits code review/verification.
4. **Closed**: Fix has been verified on the `dev` or `main` branch, and the ticket is officially closed.
